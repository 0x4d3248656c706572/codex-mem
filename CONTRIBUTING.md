# Contributing

## Principles

- keep the project Codex-focused
- prefer small, auditable changes
- avoid reintroducing multi-assistant integration code

## Local checks

Run:

```bash
node index.js version
node index.js --help
npm pack --dry-run
```

## Pull requests

Good PRs include:

- a clear problem statement
- scoped changes
- notes about install/runtime impact
- verification steps
